# 천국의 계단 (Stairway to Heaven)

Vanilla HTML/CSS/JS와 Firebase Realtime Database를 사용하는 멀티플레이 계단 오르기 게임.

## 기술 스택
- 프론트엔드: Vanilla JavaScript, HTML5, CSS3
- 서버/DB: Firebase Realtime Database
- 배포: GitHub Pages

## 핵심 설계 규칙

### 맵 코드 체계
맵은 숫자 코드 배열로 표현한다:
- `0` = 좌 1칸
- `1` = 우 1칸
- `2` = 좌 점프 (2칸)
- `3` = 우 점프 (2칸)

### 상태 관리
모든 게임 상태는 단일 `state` 객체 하나로 관리한다. 전역 변수를 흩뿌리지 않는다.

### 게임 루프
타이머는 `setInterval` 남발 금지. `requestAnimationFrame` 기반 게임 루프 하나만 사용한다.

### 시간 저장 방식
타이머 값은 카운트다운 숫자가 아니라 **만료 시각**(`Date.now()` 기준 절대값)으로 저장한다.
예: `state.roundEndsAt = Date.now() + 30000`
남은 시간 계산: `state.roundEndsAt - Date.now()`

## 코드 스타일
- 주석과 설명은 **한국어**로 작성한다.
- 변수명은 의미가 명확하게 짓는다.
- 주요 게임 로직에는 반드시 한글 주석을 달아 흐름을 설명한다.

## 개발 로드맵

7단계로 구성. 각 단계를 완료한 뒤 다음 단계로 진행한다.

**1단계 — 혼자 계단 오르기 (Firebase 없음)**
시드 기반 숫자 코드 맵 자동 생성(0=좌1칸, 1=우1칸, 2=좌점프, 3=우점프), 같은 시드면 같은 맵. 4개 키 입력(입력은 함수로 분리해 나중에 터치 버튼도 같은 걸 쓰게) → 맞으면 한 칸 상승·틀리면 추락. 높이 m 표시(계단 10개당 1m). 추락 시 좌우 1칸으로 x이동, 5초 내 계단 닿으면 그 계단부터 재개·못 닿으면 게임오버.

**2단계 — 채찍 게이지**
n초간 키 무입력 시 게이지 +1, 입력 시 -1(최소 0). n은 2000ms에서 시작해 500m에서 200ms가 되도록 선형 감소(200ms 하한). 게이지 10 도달 시 4개 키 중 랜덤 1개 발동(추락 가능). 게이지 바 표시.

**3단계 — 문제 출제**
엑셀(.xlsx) 업로드·파싱, 다 풀면 다시 섞어 반복. 30초마다 3문제, 푸는 동안 게임 진행 정지. 정답 3개=고급 카드, 2개=일반 카드, 1개=무효과, 0개=3초 정지.

**4단계 — 아이템 (한 번에 하나씩 구현)**
카드 선택 UI, 30초 후 만료(만료 시각 방식). 고급=낙하산·진정제·엘리베이터·친구따라강남·꽝·자율주행, 일반=사다리·당근·꽝·폭죽. 순간이동류(사다리·엘리베이터·친구따라강남)는 "해당 높이에서 안전한 1칸 계단을 찾아 안착"으로 통일.

**5단계 — 관리자 기능**
시작 높이 설정, 접속 클라이언트에 아이템 강제 적용, 테스트 모드, 더미 플레이어 N명 생성(부하 테스트용).

**6단계 — 멀티플레이 (Firebase)**
PIN 입장·세션 방식, 이름 검증(1~8자·빈칸 금지). 각 학생의 step·x위치·isFalling·이름을 throttle로 0.3~0.5초에 1번만 쓰기. 다른 플레이어는 우선 높이 막대기로 표시. 재접속 시 step만 복구(게이지 0·아이템 없음·문제 새로). 관리자 실시간 랭킹·게임종료 버튼(가장 높이 올라간 순 랭킹).

**7단계 — 캐릭터·연출**
네모를 이미지로 교체, 다른 플레이어도 캐릭터로(위치 보간). 폭죽·낙하산 등 시각효과, 모바일 4버튼 배치.

## 진행 상황

1단계 완료: 시드 기반 맵 생성(mulberry32), 4키 입력(handleInput 분리), 정답 시 계단 상승·오답 시 추락, 추락 후 5초 표류(좌우 1칸 이동으로 착지 재개) 또는 게임오버, 높이 m 표시.

2단계 완료: 채찍 게이지(무입력 +1·입력 -1), 높이 기반 n값 선형 감소(2000ms→200ms), 10 도달 시 랜덤 키 강제 발동, 게이지 바 표시.

2.5단계 완료(추가): 높이 구간별 난이도 맵 생성(MAP_ZONES — 점프 확률·연속성 조절), MAP_LEN 300, 카메라 X축 추적 버그 수정.

3단계 완료: xlsx 업로드·파싱(SheetJS, quiz-battle-royale 동일 양식), 30초 타이머 → 3문제 팝업·게임 정지, 5가지 유형(단답형·객관식·복수정답·순서·선잇기) 지원, 정답 수별 결과(0개=3초 freeze, 1개=무효과, 2·3개=카드 예약-4단계), 퀴즈 중 키 입력 전면 차단.

**매 작업 시작 시 이 항목과 코드를 먼저 읽고 현재 상태를 파악할 것.**
