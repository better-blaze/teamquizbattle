# 천국의 계단 (Stairway to Heaven)

Vanilla HTML/CSS/JS와 Firebase Realtime Database를 사용하는 멀티플레이 계단 오르기 게임.

## 기술 스택
- 프론트엔드: Vanilla JavaScript, HTML5, CSS3
- 서버/DB: Firebase Realtime Database
- 배포: GitHub Pages

## 핵심 설계 규칙

### 맵 코드 체계
맵은 숫자 코드 배열로 표현한다:
- `0` = 좌 1칸
- `1` = 우 1칸
- `2` = 좌 점프 (2칸)
- `3` = 우 점프 (2칸)

### 상태 관리
모든 게임 상태는 단일 `state` 객체 하나로 관리한다. 전역 변수를 흩뿌리지 않는다.

### 게임 루프
타이머는 `setInterval` 남발 금지. `requestAnimationFrame` 기반 게임 루프 하나만 사용한다.

### 시간 저장 방식
타이머 값은 카운트다운 숫자가 아니라 **만료 시각**(`Date.now()` 기준 절대값)으로 저장한다.
예: `state.roundEndsAt = Date.now() + 30000`
남은 시간 계산: `state.roundEndsAt - Date.now()`

## 코드 스타일
- 주석과 설명은 **한국어**로 작성한다.
- 변수명은 의미가 명확하게 짓는다.
- 주요 게임 로직에는 반드시 한글 주석을 달아 흐름을 설명한다.

## 개발 로드맵

7단계로 구성. 각 단계를 완료한 뒤 다음 단계로 진행한다.

**1단계 — 혼자 계단 오르기 (Firebase 없음)**
시드 기반 숫자 코드 맵 자동 생성(0=좌1칸, 1=우1칸, 2=좌점프, 3=우점프), 같은 시드면 같은 맵. 4개 키 입력(입력은 함수로 분리해 나중에 터치 버튼도 같은 걸 쓰게) → 맞으면 한 칸 상승·틀리면 추락. 높이 m 표시(계단 10개당 1m). 추락 시 좌우 1칸으로 x이동, 5초 내 계단 닿으면 그 계단부터 재개·못 닿으면 게임오버.

**2단계 — 채찍 게이지**
n초간 키 무입력 시 게이지 +1, 입력 시 -1(최소 0). n은 2000ms에서 시작해 500m에서 200ms가 되도록 선형 감소(200ms 하한). 게이지 10 도달 시 4개 키 중 랜덤 1개 발동(추락 가능). 게이지 바 표시.

**3단계 — 문제 출제**
엑셀(.xlsx) 업로드·파싱, 다 풀면 다시 섞어 반복. 30초마다 3문제, 푸는 동안 게임 진행 정지. 정답 개수에 따른 보상:

정답 3개: 카드 4~5장을 랜덤으로 제시. 이 중 3장은 일반 아이템, 1~2장은 고급 아이템으로 구성. 카드는 뒤집기 전에는 일반·고급 구분 없이 모두 똑같은 뒷면으로 보이고, 학생이 한 장을 골라 뒤집어야 어떤 아이템인지 드러남.
정답 2개: 일반 아이템 카드 3장만 제시(위와 같은 카드 선택 방식).
정답 1개: 무효과.
정답 0개: 3초간 정지.

**4단계 — 아이템 (한 번에 하나씩 구현)**
카드 선택 UI, 모든 아이템은 30초 후 만료(만료 시각 방식). 효과가 남아있는 상태에서 새 아이템을 획득하면 기존 효과는 사라지고 새 아이템으로 교체할 것인지 물어봄(중첩 안 됨). 순간이동류(사다리·엘리베이터·친구따라강남)는 "이동할 높이를 정한 뒤, 그 높이에서 안전한 1칸 계단을 찾아 안착"으로 통일.
고급 아이템:

낙하산: 추락 상태일 때 떨어지는 속도가 느려짐(추락 후 복귀를 위한 좌우 이동 시간을 벌어줌).
진정제: 채찍 게이지의 n값이 30초간 2.5배가 됨(게이지가 훨씬 천천히 참).
엘리베이터: 현재 높이보다 5m 위의 안전한 계단으로 이동.
친구따라강남: 나보다 위에 있는 플레이어 중 랜덤으로 1명의 이름이 뜨고, '예'를 누르면 그 플레이어의 높이에서 가장 가까운 안전한 계단으로 순간이동.구현은 둘로 나눔 — 순간이동·안전계단 안착 로직은 4단계에서 구현(디버그 패널에서 목표 높이를 지정해 테스트), 다른 플레이어 중 대상 선택 + 이름 표시 + '예' 선택 UI는 멀티플레이가 갖춰진 6단계에서 연결.
자율주행: 10초간 맵을 보고 올바른 키를 자동 입력해 자동으로 올라감(점프칸 포함 자동 통과).

일반 아이템:

사다리: 현재 높이보다 1m 위의 안전한 계단으로 이동.
새치기: 나보다 바로 앞 등수(step이 가장 적게 높은) 플레이어보다 2칸 위 안전한 계단으로 이동. 앞에 플레이어가 없으면 효과 없음.
당근: 채찍 게이지의 n값이 20초간 1.5배가 됨.
숨고르기: 30초간 채찍 게이지가 전혀 차지 않음. 활성화 시 타이머도 리셋해 만료 직후 즉시 게이지가 차지 않도록 함.
꽝: 아무 효과 없음(일반 카드에서 등장).
폭죽: 20초간 내가 지나가는 계단마다 폭죽 시각효과가 나타남(기능적 효과는 없는 연출용).
거울의 저주: 대상 플레이어의 화면을 3초간 좌우반전시킴. (저주 계열, 낮은 확률)
암흑의 저주: 대상 플레이어의 화면이 4초간 "정상 ↔ 매우 어둡게"를 반복하며 깜빡임. 어두운 시간은 CURSE_DARK_PERIOD 상수로 조정. (저주 계열, 낮은 확률)

저주 계열 공통 동작:
- 발동 시 "저주를 거시겠습니까?" 확인 창(5초 타이머, 초과 시 자동 취소).
- 예 → "저주가 시작됩니다." + 3초 카운트다운 후 효과 발동.
- 아니요 / 시간 초과 → 모든 플레이어 화면에 "ㅇㅇ님이 자비를 베푸셨습니다." 3초 표시.
- 저주 중에도 대상 채찍 게이지·추락은 정상 작동.
- 저주끼리 중첩 안 됨 — 진행 중이면 대기열에 쌓아 순차 처리.
- 구현 분리: 4단계에서 화면 효과를 자기 화면에 적용해 검증, 6단계에서 대상을 "나 제외 전체 플레이어"로 연결 및 "자비" 메시지 전파 구현.

각 아이템의 등장 확률은 나중에 직접 조정할 수 있도록, 일반/고급 각각 아이템별 확률(가중치)을 코드 상단에 상수(설정 객체)로 따로 모아둘 것. 카드를 뽑을 때 이 확률표를 참조하도록 구현.

**5단계 — 관리자 기능**
시작 높이 설정, 접속 클라이언트에 아이템 강제 적용, 테스트 모드, 더미 플레이어 N명 생성(부하 테스트용).

**6단계 — 멀티플레이 (Firebase)**
PIN 입장·세션 방식, 이름 검증(1~8자·빈칸 금지). 각 학생의 step·x위치·isFalling·이름을 throttle로 0.3~0.5초에 1번만 쓰기. 다른 플레이어는 계단 위 캐릭터로 표시(이미 구현됨), 받은 위치 사이는 부드럽게 보간(스르륵 이동) 처리. 재접속 시 step만 복구(게이지 0·아이템 없음·문제 새로). 관리자 실시간 랭킹·게임종료 버튼(가장 높이 올라간 순 랭킹). 멀티플레이 의존 아이템(친구따라강남의 대상 선택, 거울의 저주·암흑의 저주의 "나를 제외한 모든 플레이어" 적용)을 이 단계에서 연결.

**7단계 — 캐릭터·연출**
네모를 이미지로 교체, 다른 플레이어도 캐릭터로(위치 보간). 폭죽·낙하산 등 시각효과, 모바일 4버튼 배치.

## 진행 상황

1단계 완료: 시드 기반 맵 생성(mulberry32), 4키 입력(handleInput 분리), 정답 시 계단 상승·오답 시 추락, 추락 후 5초 표류(좌우 1칸 이동으로 착지 재개) 또는 게임오버, 높이 m 표시.

2단계 완료: 채찍 게이지 + 연출.

3단계 완료: 퀴즈 출제 시스템(xlsx 업로드·파싱, 30초마다 3문제, 카드 선택 보상).

4단계 완료: 전 아이템 구현(낙하산·진정제·엘리베이터·친구따라강남·자율주행·사다리·새치기·당근·숨고르기·꽝·폭죽·거울의 저주·암흑의 저주). 저주 상태 머신(confirmActive→countdownActive→active→queue).

5단계 완료: 관리자 패널(시작 높이·테스트 모드·아이템 강제 적용·더미 플레이어). 디버그 패널.

6단계 완료: Firebase Realtime Database 멀티플레이. PIN 방 입장, 로그인 오버레이, 위치 throttle 동기화(400ms), 다른 플레이어 보간 렌더, 재접속 시 step 복구, 저주/자비 Firebase 전파, 친구따라강남 플레이어 목록, 호스트 실시간 랭킹+게임 종료, 최종 순위 오버레이.

**매 작업 시작 시 이 항목과 코드를 먼저 읽고 현재 상태를 파악할 것.**
