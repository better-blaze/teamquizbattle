# 프로젝트 개요: 팀전 게임배틀 (실시간 웹 게임)
- **타겟:** 초등학생 (4~6모둠 진행)
- **기술 스택:** 바닐라 HTML, CSS, JavaScript (프레임워크 미사용)
- **데이터베이스:** Firebase Realtime Database (실시간 동기화) 및 구글 스프레드시트 (CSV 웹 게시 기능으로 문제 파싱)
- **구조:** SPA (Single Page Application). 팝업(Modal) 절대 금지, 모든 화면 이동 및 정보 카드 열람은 부드러운 '전체 화면 전환(Full-page transition)'으로 구현.

# 1. 화면 구성 (총 4개 View)
1. **Setting View (세팅):** 3~6모둠 선택 드롭다운, '게임준비' 버튼, 방 생성/참여 인터페이스.
2. **Board View (게임상황판):** 교실 스크린용.
   - 상단 30%: 모둠 숫자에 따라 수직 분할 (이름, 점수, 아이템, 접속상태 표시).
   - 중단 30%: 문제 및 이미지 표시 (이미지 클릭 시 전체화면 일시 확대 기능).
   - 하단 40%: 정답 입력 UI 및 선 잇기 캔버스 렌더링.
3. **Client View (클라이언트):** 학생 접속용. 모둠 선택, 대진표 작성, 정답 입력, 미스터리 카드 선택.
4. **Admin View (관리자):** 교사용 통제 패널. 문제 미리보기, 다음 문제 넘기기, 5초 카운트다운 트리거, 1점 단위 점수 수동 조절, 대진표 강제 마감 버튼.

# 2. 대기실 및 대진표 로직
- 클라이언트 접속 시 100초 타이머가 작동하며 대진표를 작성함.
- **규칙:** 모둠원 간 참전 횟수 차이가 '2'를 초과할 수 없음.
- **피드백:** 규칙 위반 배치가 예상되면 해당 학생 이름 버튼이 붉은색으로 깜빡이거나 비활성화됨.
- **종료:** 100초 종료 또는 관리자의 '강제 마감' 클릭 시 빈칸은 규칙에 맞게 무작위 채워짐.
- **정보 카드:** 하단에 3개 카드(가장 쉬운 문제 번호/유형, 미스터리 번호/유형, 쉬운 문제 2개 번호) 배치. 클릭 시 전체화면 전환.
- **복구 로직:** localStorage를 활용하여 튕김 시 세팅을 건너뛰고 기존 방/모둠/점수 상태 그대로 자동 재접속됨.

# 3. 게임 코어 및 정답 판정 (메인 엔진)
- 구글 시트 데이터 형태: `[문제번호, 유형, 문제내용, 이미지URL, 정답, 만점기준초]`
- **점수 계산:** 기본 10점. '만점기준초' 초과 시 1초당 1점씩 감점. **단, 정답을 맞히면 무조건 최소 3점 보장 (하한선 로직 필수).** 오답은 0점.
- **문제 유형별 판정:**
  1. 객관식: 번호 버튼 클릭 판정.
  2. 단답형: 타이핑 후 제출. **반드시 입력값과 정답 데이터의 모든 공백을 제거(`replace(/\s+/g, '')`)하고 비교할 것.** (유사/복수 정답 허용).
  3. 선 잇기(매칭): HTML5 Canvas 사용. 
     - 데이터 파싱: 정답 데이터는 `1:one | 2:two | 3:three` 형태로 주어짐. 파이프(`|`)와 콜론(`:`)으로 파싱하여 좌우 무작위 배치.
     - 캔버스 CSS에 모바일 스크롤 방지를 위해 `touch-action: none;` 반드시 적용. 화면 크기 변화에 대응하는 반응형 좌표 계산 필수.

# 4. 아이템 및 미스터리 카드
- **아이템 (모둠당 1개 소지, 1회 사용):**
  1. 점수 1.5배: 정답 시 획득 점수 1.5배.
  2. 저주: 타 팀 정답 입력칸 1.5~2.5초 지연 등장 (이 지연 시간도 감점 타이머에 포함됨).
  3. 지우개: 객관식(오답 버튼 1개 회색 비활성화), 선잇기(마지막 연결된 틀린 선 1개 삭제), 사용 후 타이머 2초 보상(상쇄) 처리.
- **미스터리 카드 (특정 문제 정답 시 8종 중 택 1, 상황판 순차 공개):**
  1. 패자의 역습 (꼴등 +10, 꼴등-1 +8, 꼴등-2 +2 / 15%)
  2. 아이템 회복 (사용한 아이템 1개 회복 / 10%)
  3. 흡수 (1등에게 3점, 2등에게 2점, 나머지 1점 강탈. 0점 모둠에겐 못 뺏음. **공동 등수 발생 시 해당 등수의 모든 모둠에게서 뺏음** / 10%)
  4. 쉴드 (저주 1회 방어 / 10%)
  5. 주사위 벼락 (주사위 모둠 -10점. 참여 모둠 수(3~6)에 따라 없는 번호는 무효 처리 / 15%)
  6. 꽝 (40%)

# 5. 기타 필수 구현
- 각 클라이언트/관리자/상황판 화면에 정답/오답/아이템/카운트다운 사운드 효과 구현.
- 모든 화면에 '음소거' 토글 버튼 배치.
- 관리자 화면에서 수동으로 '게임 끝내기' 시 3등->2등->1등 순차 축하 애니메이션(시상식) 출력.
