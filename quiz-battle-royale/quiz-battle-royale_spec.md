# 프로젝트 개요: Quiz Battle Royale (실시간 웹 게임)
- **타겟:** 초등학생 (4~6명 개인전 진행)
- **기술 스택:** 바닐라 HTML, CSS, JavaScript (프레임워크 미사용)
- **데이터베이스:** Firebase Realtime Database (실시간 동기화)
- **문제 파싱:** .xlsx 파일 업로드 → FileReader API로 파일을 읽고 → SheetJS(CDN) 라이브러리로 표 데이터로 변환
- **구조:** SPA (Single Page Application). 팝업(Modal) 절대 금지, 모든 화면 이동은 부드러운 '전체 화면 전환(Full-page transition)'으로 구현.

---

# 1. 화면 구성 (총 4개 View)

## 1-1. Setting View (세팅)
- 플레이어 수 선택 드롭다운: 4~6명
- 방 코드 직접 입력 필드: **알파벳+숫자 4자리 조합** (예: AB3K). Admin과 클라이언트가 동일한 코드를 입력하여 같은 방에 입장.
- .xlsx 파일 업로드 버튼 (문제 데이터 로드). 업로드 완료 시 문제 수 및 미리보기 표시.
- '게임준비' 버튼

## 1-2. Board View (게임상황판) — 교실 스크린용
- **상단 40%:** 문제 내용 및 이미지 표시. 이미지 클릭 시 전체화면 일시 확대.
- **하단 60%:** 플레이어 수에 따라 수직 분할.
  - 각 칸: 이름(P1~P6), 접속 상태, 응답 제출 여부 표시.
  - **응답 내용 공개 조건:** 3명 이상의 플레이어가 **정답을** 제출했을 때 모든 플레이어의 응답 내용을 공개. 그 전까지는 '제출 완료' 여부만 표시.

## 1-3. Client View (클라이언트) — 학생 접속용
- 방 코드 입력 후 P1~P6 중 자신의 번호 선택 (닉네임 입력 없음, 고정 이름 사용).
- 문제 유형별 정답 입력 UI 표시.
- **제출 후 UI 잠금:** 정답/오답 여부와 무관하게 한 번 제출하면 재시도 불가, 입력 UI 비활성화.

## 1-4. Admin View (관리자) — 교사용 통제 패널
- 다음 문제로 수동 이동 버튼 (관리자가 직접 눌러야만 넘어감)
- 문제 건너뛰기 버튼 (순서를 건너뛸 수 있음)
- 현재 문제 번호 / 전체 문제 수 표시
- 5초 카운트다운 트리거 버튼 (모든 화면에 카운트다운 동기화됨)
- 현재 플레이어별 제출 현황 표시
- '게임 끝내기' 버튼 → 축포 애니메이션 + 효과음 출력 (순위 표시 없음)

# 2. 게임 코어 및 정답 판정 (메인 엔진)

## 2-1. 엑셀 데이터 형태
| 문제번호 | 유형 | 문제내용 | 이미지URL | 정답 |
|---|---|---|---|---|
| 1 | 객관식 | 문제 텍스트 | (URL 또는 빈칸) | 정답 텍스트 |

- 유형 값: `객관식`, `단답형`, `선잇기`, `복수정답`, `순서`

## 2-2. 순위 판정
- 점수 없음. **정답을 맞힌 순서로만 순위를 매김.**
- 기준: **Firebase 서버 타임스탬프 도착 순** (클라이언트 시간 기준 아님).
- 정답을 맞히는 데 걸린 시간(카운트다운 종료 시점부터)을 함께 표시.
- **타이머 시작 시점:** 5초 카운트다운이 완전히 끝난 시점(0이 된 순간)부터 측정.
- **동점 처리:** 서버 도착 시간이 완전히 동일할 경우 공동 순위로 인정.
- 오답 제출 시 순위 없음(미응답 처리와 동일). 재시도 불가.

# 3. 문제 유형별 판정

## 3-1. 객관식
- 보기 개수만큼 **10~99 사이의 난수**를 뽑아 각 보기 앞에 표시 (예: `37. 사과`, `82. 바나나`).
- **난수 동기화:** 문제 시작 시점에 난수를 Firebase에 저장하여 모든 클라이언트가 동일한 번호를 보도록 함.
- 클라이언트는 정답 보기 앞의 숫자를 타이핑하여 제출. (1~4 찍기 방지 목적)

## 3-2. 단답형
- 타이핑 후 '제출' 버튼 클릭.
- **반드시 입력값과 정답 데이터의 모든 공백을 제거(`replace(/\s+/g, '')`)하고 비교.**
- 복수 정답 허용 (정답 데이터에 `|`로 구분하여 저장 가능).

## 3-3. 선 잇기 (매칭)
- HTML5 Canvas 사용.
- **데이터 파싱:** 정답 데이터는 `1:one | 2:two | 3:three` 형태. 파이프(`|`)와 콜론(`:`)으로 파싱하여 좌우 무작위 배치.
- 캔버스 CSS에 모바일 스크롤 방지를 위해 `touch-action: none;` 반드시 적용.
- 화면 크기 변화에 대응하는 반응형 좌표 계산 필수.
- **모든 항목을 연결해야만 '제출' 버튼이 활성화됨.**

## 3-4. 복수정답
- **데이터 파싱:** 정답 데이터는 `1,4` 형태 (쉼표로 구분).
- 보기 앞에 난수 번호를 붙이지 않음. 보기를 있는 그대로 표시.
- 해당하는 항목을 모두 체크한 후 '제출' 버튼 클릭 시 정답 판정.

## 3-5. 순서
- **데이터 파싱:** 정답 데이터는 `1,4,3,2` 형태 (올바른 순서).
- 클라이언트는 보기를 순서대로 클릭하며 선택.
- **선택 취소:** 이미 선택한 항목을 다시 클릭하면 해당 항목부터 이후에 선택된 항목 전체가 취소됨.
- 올바른 순서로 모두 선택한 뒤 '제출' 버튼 클릭 시 정답 판정.

# 4. 엑셀 파싱 구현 방식
- **FileReader API:** 브라우저 내장 기능. 사용자가 업로드한 `.xlsx` 파일을 바이트로 읽어옴.
- **SheetJS (CDN):** 읽어온 바이트를 2차원 배열(표)로 변환해주는 라이브러리. HTML에 `<script>` 한 줄로 불러옴.
- 파싱 흐름: 파일 선택 → FileReader가 읽음 → SheetJS가 표로 변환 → JavaScript 배열로 Firebase에 업로드.
- 업로드 위치: **Setting View**에서 처리.

---

# 5. 기타 필수 구현
- **사운드 효과:** 정답, 오답, 카운트다운 사운드를 클라이언트/관리자/상황판 모든 화면에 적용.
- **음소거 토글 버튼:** 모든 화면에 배치.
- **시상식:** 관리자 화면에서 '게임 끝내기' 클릭 시 축포 애니메이션과 효과음 출력 (순위 표시 없음).
- **정답 공개 없음:** 교사가 구두로 정답을 설명. 시스템에서 정답을 공개하지 않음.
- **문제 진행:** 기본적으로 엑셀의 모든 문제를 순서대로 진행. 관리자가 건너뛰기 가능.
- **복구 로직 없음:** 접속이 끊기면 재접속 후 방 코드와 플레이어 번호를 다시 선택하여 입장.
